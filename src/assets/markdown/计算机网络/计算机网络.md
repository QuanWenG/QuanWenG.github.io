1. 网络分层模式 OSI七层模型
- 应用层 为计算机用户提供服务
- 表示层 数据处理
- 会话层 管理应用会话
- 传输层 为两台主机之间提供数据传输服务
- 网络层 路由和寻址
- 数据链路层 帧编码和纠正
- 物理层 比特流传输

2. TCP/IP 四层模型
- 应用层 OSI 1-3
- 传输层 OSI 4
- 网络层 OSI 5
- 网络接口层 OSI 6-7

3. 网络为什么分层
- 各层独立
- 高灵活和可替换
- 简化问题

4. 应用层协议
- HTTP 超文本传输协议 *
- SMTP 简单邮件发送协议
- POP3/IMAP 邮件接收协议
- FTP 文件传输协议 *
- Telnet 远程登陆协议
- SSH 安全的网络传输协议 *
- RTP 实时传输协议
- DNS 域名管理系统 *

5. 传输层协议 
- TCP 传输控制协议 面向链接 可靠 
- UDP 用户传输协议 无连接 尽最大努力

6. 网络层协议
- IP 网际协议
- ARP 地址解析协议
- ICMP 互联网控制报文
- NAT 网络地址转换协议
- OSPF 开放式最短路径优先协议
- RIP 路由信息协议
- BGP 边界网关协议

7. 从输入url到页面展示发生什么
- 浏览器查找域名的IP地址 DNS
- 浏览器向web服务器发送http请求
- 服务器处理请求
- 服务器返回HTML响应
- 浏览器渲染HTML

8. HTTP和HTTPS 有什么区别
- HTTP端口80 HTTPS端口443
- URL前缀 http:// https://
- 安全性和资源消耗 http不安全但消耗少 https安全但消耗多
- SEO（搜索引擎优化） https浏览器优先显示

9. GET和POST区别
- 语义不同: GET用于获取和查询资源 POST用于创建和修改资源
- 格式不同: GET请求参数通常放在URL中,POST请求参数通常放在请求体中
- 安全性不同: GET请求因为将参数放在URL中,更容易泄露敏感数据

10. 什么是webSocket
- WebSocket是一种基于TCP连接的全双工通信协议，客户端和服务器可以同时发送和接收数据
- 弥补http在持久通信的不足

11. WebSocket和http的区别
- WebSocket 双向实时通信协议
- HTTP 单向通信协议
- WebSocket 支持扩展
- WebSocket 通信数据格式轻量

12. WebSocket的工作流程
- 客户端发送一个http请求要求升级为ws
- 服务器收到后，进行升级操作，返回101
- 客户端和服务器建立一个ws连接，进行双向数据传输
- 客户端和服务器可以主动发送关闭帧，对方收到回回复关闭帧，关闭TCP连接

13. WebSocket和长轮询和短轮询的区别
- 短轮询 每隔固定时间发起http请求，服务器收到后响应
- 短轮询好处 简单 兼容
- 短轮询缺点 实时性一般 资源浪费大
- 长轮询 服务器暂无数据会保持连接，直到超时或者有新数据
- 长轮询优点 及时 减少空响应
- 长轮询缺点 资源占用高 资源浪费大
- ws 通过一次http握手后，建立持久的TCP连接
- ws优点 实时 资源效率高 功能强
- ws缺点 需要支持ws 相比轮询实现麻烦

14. SSE与WebSocket有什么区别
- SSE 单向通信 只有服务器向客户端发送数据
- SSE 基于http/https实现
- SSE 实现相对简单
- SSE 原生支持
- SSE 主要设计传输文本
- SSE常用于大模型api的流式响应
