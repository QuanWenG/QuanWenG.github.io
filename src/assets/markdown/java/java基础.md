Java 是一门静态类型、面向对象的通用编程语言。它通常先编译为 JVM 字节码，再由具体平台上的 Java 虚拟机执行，因此同一份字节码可以跨平台运行。

## Java 平台与执行方式

### Java SE、Jakarta EE、JDK 与 JVM

- **Java SE** 提供语言、标准类库和 JVM 规范等基础能力。
- **Jakarta EE**（原 Java EE）是在 Java SE 之上的企业应用规范集合，例如 Servlet、JPA 和 Jakarta REST。Spring 是独立生态，不属于 Jakarta EE 规范；JDBC 属于 Java SE API。
- **JDK** 是开发工具包，包含编译器、调试与诊断工具，以及运行 Java 程序所需的组件。
- **JVM** 负责加载、验证和执行字节码。现代 JDK 通常直接提供运行时镜像，不一定再单独发行传统意义上的 JRE。

典型执行链路如下：

```text
.java 源文件 → javac 编译 → .class 字节码 → JVM 解释执行 / JIT 编译 → 本机机器码
```

### 解释执行、JIT 与 AOT

JVM 可以解释字节码，也可以用即时编译器（JIT）把运行中的热点代码编译为机器码。JIT 能利用真实运行信息优化代码，但需要一定的预热时间。

预先编译（AOT）在程序运行前生成本机代码，通常能缩短启动时间并降低预热成本；代价是构建更复杂、产物与目标平台相关，而且反射、动态代理和运行时生成类等特性可能需要额外配置。

### OpenJDK 与 Oracle JDK

OpenJDK 是 Java SE 的开源参考实现。Oracle JDK 基于 OpenJDK 构建，两者在兼容性上非常接近，主要差异通常落在发行节奏、许可、商业支持和附带工具上。选择时应查看所用版本的许可证与支持周期，而不是把差异简单概括为“免费”或“收费”。

## 类型、变量与运算

### 基本类型

| 类型 | 位数 | 取值或说明 |
| --- | ---: | --- |
| `byte` | 8 | -2⁷ ～ 2⁷-1 |
| `short` | 16 | -2¹⁵ ～ 2¹⁵-1 |
| `int` | 32 | -2³¹ ～ 2³¹-1 |
| `long` | 64 | -2⁶³ ～ 2⁶³-1 |
| `char` | 16 | UTF-16 代码单元，0 ～ 2¹⁶-1 |
| `float` | 32 | IEEE 754 单精度浮点数 |
| `double` | 64 | IEEE 754 双精度浮点数 |
| `boolean` | 未规定存储位数 | 语言层面只有 `true` 和 `false` |

引用类型变量保存对象引用。对象一般由 JVM 管理内存，其具体布局、逃逸分析结果及对象是否实际分配到堆上属于虚拟机实现细节，不应机械地理解为“基本类型一定在栈、包装类型一定在堆”。

### 包装类与自动装箱

八种基本类型都有对应包装类。自动装箱和拆箱让基本值与包装对象之间可以自动转换，但仍需注意：

- 包装对象可能为 `null`，拆箱时可能抛出 `NullPointerException`。
- 基本类型的 `==` 比较数值；引用类型的 `==` 比较是否为同一引用。
- 包装类的 `equals` 通常比较同类型的值。
- 部分装箱值必须被缓存，但不要依赖所有包装类型、所有数值范围都共享对象。

### 成员变量、局部变量与 `static`

- 实例变量属于对象；类变量使用 `static` 修饰，由该类共享。
- 成员变量具有默认初始值；局部变量在读取前必须明确赋值。
- 静态方法没有隐式的 `this` 接收者，因此不能直接访问某个对象的实例成员；拿到对象引用后仍可通过该引用访问。

### 常见运算与控制语句

- `++a` 先自增再参与表达式；`a++` 先使用原值再自增。
- `continue` 跳过本轮循环，`break` 结束当前循环或 `switch`，`return` 结束当前方法并可返回结果。
- `>>` 是带符号右移，`>>>` 是无符号右移。
- 对 `int` 的移位距离只使用低 5 位，对 `long` 只使用低 6 位，因此 `x << 42` 对 `int` 等价于 `x << 10`。
- 位运算适合位掩码、协议字段、哈希与部分编码算法，但其性能应以实际基准为准，不能笼统断言“一个时钟周期完成”。

## 方法与面向对象

### 重载、重写与可变参数

- **重载**：同一作用域内方法名相同、参数列表不同；仅返回类型不同不能构成重载。
- **重写**：子类为可继承的实例方法提供新实现，方法签名及返回类型需满足语言规则。
- 可变参数写作 `String... args`，在方法内部按数组处理，并且必须位于参数列表末尾。

### 构造方法

构造方法与类同名且没有返回类型，用于初始化对象。类未声明任何构造方法时，编译器才会生成默认无参构造方法；一旦显式声明任意构造方法，就不会再自动生成它。构造方法可以重载，但不能被继承或重写。

### 封装、继承、多态与抽象

- **封装**：隐藏实现细节，通过稳定接口维护对象约束。
- **继承**：子类继承父类中可继承的成员，但不能直接访问父类的 `private` 成员，构造方法也不会被继承。
- **多态**：父类型引用可以指向子类型对象，实例方法调用在运行时按实际对象类型分派。
- **抽象**：提炼共同能力和契约，常借助抽象类与接口表达。

抽象类适合共享状态和实现；接口适合描述能力与协作契约。现代 Java 接口除抽象方法外，还可以包含默认方法、静态方法和私有辅助方法。抽象类与接口都不能直接实例化，具体子类仍需实现尚未实现的抽象方法。

### 相等性与哈希

`Object.equals` 的默认行为与引用相等一致，类可以重写它来表达值相等。若两个对象依据 `equals` 相等，它们的 `hashCode` 必须相同；反过来，哈希值相同并不代表对象相等。因此重写 `equals` 时通常也必须重写 `hashCode`。

### 拷贝语义

- **引用拷贝**：复制引用，两个变量指向同一对象。
- **浅拷贝**：创建外层新对象，但内部可变对象仍可能共享。
- **深拷贝**：递归复制需要独立的可变状态；具体边界应由业务语义定义。

## 字符串

- `String` 不可变，适合作为键、常量和跨线程共享的值。
- `StringBuilder` 可变且不提供同步保证，适合单线程拼接。
- `StringBuffer` 的主要方法带同步，适用于确实需要共享可变缓冲区的场景。

`String` 的不可变性来自类不能被继承、内部字符存储不向外暴露且 API 不提供原地修改；不只是“字段用了 `final`”。字符串字面量会进入字符串池，以复用相同字面量。`String.equals` 已重写为按字符内容比较。

## 注释与命名

Java 支持单行注释 `//`、多行注释 `/* ... */` 和文档注释 `/** ... */`。标识符是程序员为类、方法、变量等声明的名称；关键字由语言保留，不能用作普通标识符。
