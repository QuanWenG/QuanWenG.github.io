1. java的优势
- 生态好
- 面向对象
- 并发优势
- 安全（GC自动回收）
- 跨平台

2. javaSE和javaEE的区别
- javaSE 基本语法，内部库
- javaEE javaEE指的企业级的开发标准和规范，如spring jdbc jsp等

3. JDK和JRE
- JDK 指的是java一系列开发工具包含JRE
- JRE是运行已编译的java程序所需环境
- - (包含1.JVM java虚拟机 2.java基础库)

4. 什么是字节码？字节码的好处？
- java编译后 JVM 可以理解的代码就是字节码(.class)
- 解决了传统解释型语言的效率问题，保留了解释型语言可移植的特点
-  .java -> javac编译 -> .class -> 解释器&JIT -> 机器码
- JIT编译器 运行时编译 完成编译后把机器码保存
- JIT 编译热点代码

5. 为什么说java语言 编译与解释并存
- java编译成字节码 字节码必须由java解释器 解释运行

6. AOT有什么优点？为什么不使用AOT呢？
- AOT在程序执行前静态编译
- 没有jit预热花销，启动速度快
- 提高安全性，不易反编译和修改
- 但是AOT不支持java一些动态特性 spring需要专门优化和适配

7. Oracle JDK 和 OpenJDK
- OpenJDK 完全开源 OracleJDK 不是完全开源的
- OracleJDK 会提供免费版本，但有时间限制
- OracleJDK 会提供一些功能和工具
- 协议不同

8. java和cpp比较
- 内存安全，不直接提供指针内存访问
- 单继承，多继承可用接口实现
- 内存自动回收 GC
- java只有方法重载 没有运算符重载

9. java注释有三种
- 单行注释
- 文档注释
- 多行注释

10. 标识符和关键词的区别
- 标识符就是名字
- 关键词是具有特殊的标识符

11. 自增自减运算符
- 前缀 ++a 先增后用
- 后缀 a++ 先用后增

12. 位运算符
- “>>>” 无符号右移 “>>” 右移
- 位运算高效，作用与处理器的硬件指令，一个时钟周期完成
- 节省内存，可用一个整数存储多个布尔值
- 用处: 位字段管理 哈希算法加密解密 数据压缩 数据校验 内存对齐
- 超过数值位数会先求余再移位 x<<10 == x<<42(42%32 10) 

13. continue break return的区别
- continue 跳过这次循环
- break 跳出这个循环体
- return 跳出这个函数 返回值

14. java数据类型
- byte 8位 1字节 -128~127
- short 16位 2字节 -2^15~2^15-1
- int 32位 4字节 -2^31~2^31-1
- long 64位 8字节 -2^63~2^63-1
- char 16位 2字节 0~2^16-1
- float 32位 4字节
- double 64位 8字节
- boolean 1位(未明确定义)

15. 基本类型和包装类
- 基本数据类型的局部变量在JVM中的栈中,包装类在堆中
- 基本数据类型占用空间小
- 基本类型的默认值不是null
- 基本类型 == 比较值 包装类 == 比较对象的内存地址 equals比较值

16. 包装类具有缓存机制
- 默认会有缓存机制提升性能
- 范围[-128,127]

17. 成员变量和局部变量
- 成员变量属于类，生命周期随类，存储在堆，具有默认值
- 局部变量属于方法， 生命周期随方法，存储在栈，不会自动赋值

18. 静态变量（static）
- 类中的static变量被该类的所有实例共享，节省内存

19. 静态方法为什么不能调用非静态成员
- 静态方法属于类，类加载后分配内存，可以直接访问
- 非静态成员需要等类实例化后才会分配内存，才可以访问

20. 静态方法和实例方法的区别
- 静态方法无需创建对象使用
- 静态方法只允许访问静态成员

21. 重载和重写的区别
- 重载 同一个类中，方法名相同，参数不同
- 重写 子类对于父类中同方法名，同参数的实现重写编写

22. 可变长参数
- 允许传入不定长参数 String... args
- 可变长参数只能是函数参数列表的最后一个参数

23. 面向对象和面向过程
- 面向过程编程（POP）:把解决问题的过程拆成一个个方法，通过一个个方法解决
- 面向对象编程（OOP）:面向对象会抽象出对象，用对象执行方法的方式解决问题

24. 面向对象的优点
- 易于维护，易于复用，易于扩展

25. 对象相等和引用相等
- 对象相等指内容相等 equals
- 引用相等指内存地址相等 ==

26. 类没有声明构造方法，程序能执行吗
- 构造方法是特殊的方式，用于完成对象的初始化工作
- 默认会构造无参构造器
- 如果声明了构造方法，无论有无参数，都不会自动构造无参构造器

27. 构造方法的特点
- 名称与类名相同
- 没有返回值
- 自动执行
- 不能重写，可以重载

28. 面向对象的三大特征
- 封装 继承 多态 （抽象）
- 封装: 把属性隐藏在对象内部，不允许外部对象直接访问对象内部信息
- 继承: 继承父类的所有属性和方法
- 多态: 一个对象具有多种状态，父类引用指向子类
- 抽象: 思想层面把共有的抽象出来，抽象类和接口

29. 接口和抽象的同和不同
- 都不能实例化，接口通过实现，抽象通过被继承
- 具有抽象方法，子类或实现类必须实现
- 接口主要用于对类的行为约束，抽象主要为了代码复用，强调所属

30. 深拷贝和浅拷贝和引用拷贝
- 浅拷贝 堆上新建一个新的对象，但内部对象不会拷贝
- 深拷贝 包括对象内部的所有对象都会进行拷贝
- 引用拷贝 只拷贝内存地址

31. hashCode()
- 散列码，确定该对象在哈希表中索引位置

32. 为什么说重写equals必须重写hashCode
- 相等的对象的hashCode值必须相等

33. String StringBuffer StringBuilder
- String 不可变的
- StringBuilder 可变的
- StringBuffer 加了同步锁 线程安全的 可变的

34. String为什么不可变
- String类使用final关键字保存字符串

35. String的equals和Object的equals区别
- String内部equals被重写过
- Object内部equals比较对象内存地址

36. 字符串常量池
- JVM为了提升性能减少内存消耗针对String开辟的独立空间，避免字符串重复创建
