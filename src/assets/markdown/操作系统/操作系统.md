操作系统负责管理处理器、内存、存储和外设，并向应用提供进程、文件、网络等抽象。内核是操作系统的核心部分，但操作系统还包括系统服务、命令行工具和图形界面等组件。

## 主要职责

- **进程与线程管理**：创建、调度、同步和终止执行单元。
- **内存管理**：分配物理内存，提供虚拟内存和地址空间隔离。
- **文件与存储管理**：组织持久化数据，维护权限、缓存和一致性。
- **设备管理**：通过驱动程序控制设备并处理中断。
- **网络管理**：实现协议栈、套接字和网络设备抽象。
- **安全管理**：进行身份、权限、隔离与审计。

常见操作系统家族包括 Windows、GNU/Linux、各类 Unix 与 BSD、macOS，以及面向不同设备形态的 HarmonyOS 等。

## 用户态与内核态

现代处理器通常提供不同特权级。应用代码主要运行在较低特权的**用户态**，不能随意执行特权指令或访问任意内核地址；内核运行在较高特权的**内核态**，可以管理页表、设备、中断和调度器等关键资源。

这种隔离的目标是限制故障与攻击的影响范围：单个应用崩溃通常不会直接破坏内核或其他进程。用户态并不等于“只能读取自己的数据”，实际可访问范围由虚拟内存映射、操作系统权限和进程间通信机制共同决定。

## 特权级切换

用户态代码需要内核服务时，会通过受控入口进入内核。常见原因包括：

1. **系统调用**：应用主动请求文件、网络、进程或内存服务。
2. **中断**：计时器或设备向处理器发出异步事件。
3. **异常**：执行中发生缺页、除零、非法指令等同步事件。

进入内核后，处理器保存必要的执行现场并转到预先注册的处理入口；处理完成后恢复现场并返回原特权级。一次系统调用不一定引发线程上下文切换，但特权级转换、参数检查及缓存影响都会带来成本，因此高频小操作常通过批量处理、缓冲或异步 I/O 降低开销。

## 进程、线程与调度

- **进程**是资源与地址空间的主要隔离单位。
- **线程**是常见的 CPU 调度执行单位，同一进程内线程通常共享地址空间和大部分资源。
- **上下文切换**会保存当前执行状态并恢复另一个线程或进程的状态，可能影响 CPU 缓存和地址转换缓存。

并发不等于并行：单核处理器可以通过交替调度实现并发，多核处理器才可能让多个执行流在同一时刻并行运行。

## 虚拟内存

虚拟内存为每个进程提供独立的虚拟地址空间，再由页表映射到物理内存或其他后备存储。它带来地址隔离、按需分配、共享映射等能力。访问尚未建立有效映射的页面会触发缺页异常；缺页是机制，不一定意味着必须从磁盘读取数据。
