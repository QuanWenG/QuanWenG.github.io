1. 什么是redis
- 基于c开发数据存储在内存中，内存读写的NoSQL数据库

2. redis为什么快
- 纯内存操作
- 高效的I/O模型
- 优化的内部数据结构
- 简洁高效的通信协议

3. 为什么使用Redis
- Redis基于内存，访问速度快
- Redis支持高并发
- Redis功能全面，可做分布式锁，限流，消息队列，延时队列

4. 为什么用Redis而不用本地缓存
- 数据一致性,多服务器部署本地缓存不一致
- 内存限制，本地受限于单一服务器内存
- 数据丢失风险，本地服务器宕机数据丢失
- 易于维护，集中管理
- 功能丰富，支持多种数据结构和功能

5. Redis做消息队列
- redis2.0引入了发布订阅功能，解决了list实现消息队列没有广播机制，消息只能消费一次的问题
- 发布者通过 publish 投递消息给指定的 channel
- 订阅者通过 subscribe 订阅关心的 channel

6. 如何基于Redis实现延时任务
- Redis 过期事件监听 失效差 丢消息 多服务实例重复消费 不推荐
- Redisson 内置的延时队列
- redisson优势 减少了丢消息的可能（也可能丢）
- redisson优势 消息不存在重复消息问题 从同一目标队列获取任务

7. redis线程模型
- 单线程模式
- 4.0 大键值对的异步删除操作
- 6.0 多线程处理网络请求

8. redis单线程模型
- 基于Reactor模式的事件处理模型
- 通过IO多路复用技术实现监听大量客户端连接

9. redis之前为什么单线程
- 单线程更易维护
- Redis性能瓶颈不在cpu
- 多线程存在死锁，线程上下文切换，影响性能

10. 为什么后来又引入多线程
- 提供网络IO读写性能
